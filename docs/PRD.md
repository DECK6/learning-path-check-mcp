# PRD: 우리 아이 뭐 배우지? 체크

- 문서 버전: v0.2
- 제품 유형: PlayMCP 공개 MCP
- 가칭 MCP ID: `learningPathCheck`
- 핵심 자산: `korean-elementary-learning-map`, `korean-secondary-learning-map`, DEXA AI School 벤치마크

## 1. 제품 개요

**우리 아이 뭐 배우지? 체크**는 국가교육과정 온톨로지를 바탕으로 아이가 배우는 내용, 필요한 선수개념, 막힌 지점, 복습 순서와 학습 변화를 학부모에게 근거와 함께 알려주는 학습 내비게이터다.

일반 AI의 추측이 아니라 DECK이 구축한 초·중·고 교육과정 온톨로지와 성취기준 관계를 결정론적으로 탐색한다.

제품은 다음 질문에 답한다.

1. 우리 아이가 지금 무엇을 배우는가
2. 이 내용을 이해하려면 무엇을 먼저 알아야 하는가
3. 어디서 막혔고 어떤 순서로 점검해야 하는가
4. 지난 점검 이후 무엇이 달라졌는가
5. 다음에는 언제, 무엇을 다시 확인해야 하는가

## 2. 문제 정의

### 사용자 문제

학부모는 아이가 특정 단원을 어려워해도 원인이 현재 단원인지 이전 개념인지 알기 어렵다. 교육과정 문서는 정확하지만 일반 사용자가 학습 경로를 파악하기 어렵고, 일반 AI는 교육과정 적용 시기·학년·성취기준·선수관계를 혼동할 수 있다.

기존 AI 학습 서비스는 문제은행이나 자사 콘텐츠 사용 이력을 전제로 하는 경우가 많다. 학부모가 일상 언어로 아이의 상황을 말하고, 교육과정 근거를 확인하며, 점검 결과와 다음 행동까지 계속 관리할 수 있는 대화형 도구가 필요하다.

### 해결 방식

1. 자연어 질문을 교육과정의 과목·영역·성취기준·세부 주제로 연결한다.
2. 그래프를 탐색해 선수개념과 후속 학습을 반환한다.
3. 아이가 어렵다고 말한 사실만으로 결손을 단정하지 않고 최소 확인 질문을 제시한다.
4. 답변을 바탕으로 `이해함 / 복습 필요 / 추가 확인 필요`로 판정한다.
5. 판정 결과를 자녀별 학습 프로필에 기록한다.
6. 복습 계획, 다음 점검일, 변화 추이를 학부모 리포트로 제공한다.

## 3. 목표 사용자

### 1차 사용자

- 초·중·고 자녀를 둔 학부모
- 자녀의 학습 상태와 복습 우선순위를 관리하려는 보호자

### 2차 사용자

- 자신의 학습 순서를 확인하려는 학생
- 과외·학원 교사
- 학교 교사
- 교육 콘텐츠 기획자

### 대표 질문

- “초5 아이가 분수 나눗셈을 어려워해요. 어디부터 확인해야 해?”
- “중2 2학기 수학에서 뭘 배우고 지금 뭘 준비해야 해?”
- “함수를 배우기 전에 어떤 개념을 알아야 해?”
- “지난번에 약분이 약하다고 했는데 지금은 나아졌어?”
- “이번 주에 20분씩 할 복습 계획 만들어줘.”
- “다음 주에 다시 확인할 내용을 알려줘.”
- “이번 달 학습 상태를 학부모용으로 요약해줘.”

## 4. 제품 목표

### P0 목표

P0는 단발성 교육과정 조회 도구가 아니라 자녀별 학습 점검과 복습을 계속 관리하는 학부모용 제품으로 출시한다.

- 초·중·고 교육과정 탐색
- 선수·후속 학습 경로 추적
- 자녀 프로필 생성과 관리
- 학습 결손 확인 질문
- 답변 기반 학습 상태 판정
- 점검 결과와 학습 이력 저장
- 자녀별 복습 계획
- 다음 점검일과 예정 항목 관리
- 이전 점검 대비 변화 확인
- 학부모용 주간·월간 리포트
- 저장 데이터 조회·수정·삭제

### P0 제외 범위

- 문제 사진 자동 채점
- 서술형 답안 정밀 평가
- 학생 성적이나 입시 결과 예측
- 학원·교재·유료 상품 추천
- 대학 합격 가능성 판단
- 교사용 학급 단위 대시보드
- 학교 공식 성적표 연동
- 생성형 AI가 임의로 만든 교육과정 관계

## 5. 핵심 사용자 흐름

### 흐름 1: 자녀 프로필 시작

1. 학부모가 자녀의 학교급·학년·관심 과목을 말한다.
2. MCP가 최소 정보로 자녀 프로필을 생성한다.
3. 학부모가 원하는 경우 별명, 학습 목표, 주당 학습 가능 시간을 추가한다.
4. 서버는 사용자별로 프로필을 격리 저장한다.

### 흐름 2: 학년별 학습 내용 확인

1. 사용자가 학년·과목을 질문한다.
2. MCP가 해당 학교급과 과목을 식별한다.
3. 영역·주제·성취기준을 구조화해 반환한다.
4. 현재 자녀 프로필이 있으면 해당 학년과 기존 점검 이력을 함께 반영한다.
5. 특정 개념을 선택하면 선수·후속 경로를 확장한다.

### 흐름 3: 어려움의 원인 점검

1. 학부모가 아이가 어려워하는 내용을 설명한다.
2. MCP가 목표 개념 후보를 찾는다.
3. 모호하면 후보를 제시해 확인한다.
4. 선수개념 그래프를 역추적한다.
5. 우선 확인할 질문을 생성한다.
6. 학부모 또는 아이가 답한다.
7. MCP가 개념별 상태를 판정하고 근거를 설명한다.
8. 점검 결과를 자녀 프로필에 기록한다.
9. 다음 복습 행동과 재점검일을 제안한다.

### 흐름 4: 복습 계획과 재점검

1. MCP가 복습 필요 개념을 선수관계에 따라 정렬한다.
2. 학부모가 가능한 기간과 하루 학습 시간을 정한다.
3. MCP가 주차·일차별 복습 계획을 만든다.
4. 완료한 활동과 어려웠던 항목을 기록한다.
5. 예정된 재점검일에 같은 개념을 다시 확인한다.
6. 이전 결과와 현재 결과의 변화를 보여준다.

### 흐름 5: 학부모 리포트

1. 학부모가 주간 또는 월간 요약을 요청한다.
2. MCP가 점검·복습·재점검 이력을 불러온다.
3. 다음 내용을 요약한다.
   - 확인한 개념
   - 이해된 개념
   - 복습 중인 개념
   - 아직 정보가 부족한 개념
   - 지난 기간 대비 변화
   - 다음 기간의 우선순위
   - 예정된 재점검
4. 평가나 낙인 대신 관찰된 변화와 다음 행동을 중심으로 표현한다.

## 6. P0 MCP 도구

PlayMCP 노출 도구는 10개 이내로 유지한다.

### 1. `manage_child_profile`

자녀 프로필을 생성·수정·조회·삭제한다.

주요 입력:

```json
{
  "action": "create",
  "childId": null,
  "nickname": "첫째",
  "schoolLevel": "elementary",
  "grade": 5,
  "interestedSubjects": ["수학"],
  "learningGoals": ["분수 단원 보완"],
  "minutesPerDay": 20
}
```

저장 범위:

- 자녀 별명
- 학교급과 학년
- 관심 과목
- 학습 목표
- 학습 가능 시간

학교명·실명·연락처는 요구하지 않는다.

### 2. `search_curriculum`

자연어로 과목·영역·성취기준·세부 주제를 검색한다.

출력:

- 개념 ID
- 표시 이름
- 학교급·학년군·과목
- 관련 성취기준 코드
- 검색 일치 이유
- 데이터 버전

### 3. `get_curriculum_overview`

특정 학교급·학년·과목에서 배우는 내용을 보여준다.

현재 자녀 프로필과 이력이 있으면 이미 점검한 내용과 앞으로 확인할 내용을 함께 표시한다.

### 4. `trace_learning_path`

목표 개념의 선수·후속 학습 경로를 추적한다.

출력:

- 목표 개념
- 선수개념 경로
- 후속 학습 경로
- 학교급 전이
- 관계 종류
- 연결 성취기준
- 관계 근거와 데이터 버전

### 5. `create_learning_check`

목표 개념을 이해하기 위해 필요한 최소 점검 문항을 생성한다.

```json
{
  "childId": "child-001",
  "conceptId": "concept-id",
  "observedDifficulty": "분수 나눗셈 문제에서 자꾸 틀림",
  "itemCount": 5
}
```

출력:

- 점검 ID
- 점검 대상 선수개념
- 질문 ID
- 확인 질문
- 기대되는 핵심 요소
- 판정 기준
- 질문 순서의 이유

### 6. `assess_learning_check`

점검 답변을 판정하고 결과를 자녀별 이력에 저장한다.

```json
{
  "childId": "child-001",
  "checkId": "check-001",
  "responses": [
    {
      "questionId": "q1",
      "response": "사용자 답변"
    }
  ]
}
```

출력:

- 개념별 상태: `understood`, `review_needed`, `needs_more_info`
- 판정 근거
- 가장 먼저 복습할 지점
- 목표 개념까지의 복귀 경로
- 이전 점검 결과와의 차이
- 권장 재점검일

### 7. `build_review_plan`

판정 결과를 실행 가능한 복습 계획으로 만들고 저장한다.

```json
{
  "childId": "child-001",
  "targetConceptId": "concept-id",
  "reviewConceptIds": ["concept-a", "concept-b"],
  "durationWeeks": 2,
  "minutesPerDay": 20,
  "startDate": "2026-07-14"
}
```

출력:

- 계획 ID
- 주차·일차별 학습 순서
- 학습 목표
- 완료 확인 항목
- 연결 성취기준
- 목표 개념 재도전 시점
- 캘린더 등록용 일정 데이터

### 8. `record_learning_progress`

복습 활동의 완료 여부와 학부모 관찰을 기록한다.

```json
{
  "childId": "child-001",
  "planId": "plan-001",
  "conceptId": "concept-a",
  "status": "completed",
  "observation": "통분은 혼자 했지만 약분에서 한 번 도움을 요청함",
  "recordedAt": "2026-07-16"
}
```

### 9. `get_upcoming_learning_actions`

앞으로 해야 할 복습과 재점검을 날짜순으로 조회한다.

출력:

- 예정된 복습 활동
- 재점검 예정 개념
- 지연된 활동
- 오늘 할 일
- 캘린더 등록용 데이터

### 10. `get_parent_learning_report`

자녀별 학습 상태와 변화를 학부모용으로 요약한다.

```json
{
  "childId": "child-001",
  "period": "monthly",
  "from": "2026-07-01",
  "to": "2026-07-31"
}
```

출력:

- 기간 요약
- 이해된 개념
- 복습 필요 개념
- 정보 부족 개념
- 완료한 학습 활동
- 이전 기간 대비 변화
- 다음 우선순위
- 예정된 재점검
- 관련 성취기준

## 7. 응답 설계

### 기본 표시 순서

1. 한 줄 결론
2. 지금 확인하거나 복습할 개념
3. 왜 이 개념을 먼저 보는지
4. 관련 성취기준
5. 이전 점검 대비 변화
6. 다음 행동과 예정일
7. 데이터 버전과 근거

### 표현 원칙

금지 표현:

- “아이는 이 개념을 모릅니다.”
- “학습부진입니다.”
- “이 학년 수준에 미달합니다.”

권장 표현:

- “현재 답변에서는 이 개념을 한 번 더 확인하는 것이 좋습니다.”
- “지난 점검보다 스스로 설명한 항목이 늘었습니다.”
- “추가 답변이 있어야 판단할 수 있습니다.”
- “목표 개념 전에 다음 내용을 점검해 보세요.”

## 9. 기술 아키텍처

```text
초·중·고 교육과정 데이터
    ↓
통합 어댑터
    ↓
인메모리 검색 인덱스 + 학습 그래프
    ↓
결정론적 경로·판정 엔진
    ↓
사용자별 학습 이력 저장소
    ↓
프레젠터
    ↓
PlayMCP Streamable HTTP 서버
```

### 구성 요소

#### 교육과정 데이터 계층

- 빌드 시 초등·중등 온톨로지를 공통 모델로 컴파일
- 요청 시 외부 네트워크 없이 읽기 전용 인덱스 조회
- 데이터 버전과 출처를 모든 결과에 포함

#### 판정 엔진

- 선수개념 그래프 역추적
- 질문별 판정 규칙
- 개념 상태 집계
- 정보 부족 처리
- 이전 결과와 변화 비교

#### 사용자 데이터 계층

PlayMCP in KC 컨테이너의 로컬 파일에 의존하지 않는다.

- 외부 영속 데이터베이스 사용
- 인증 사용자 ID를 해시한 내부 키 사용
- 사용자별 데이터 격리
- 자녀 프로필·점검·계획·진행 기록 저장
- 전체 조회·수정·삭제 지원
- 원문 대화 전체는 저장하지 않고 구조화 결과만 저장

#### 프레젠터

Markdown과 `structuredContent` 양쪽에 다음 값을 동일하게 제공한다.

- `childId`
- `conceptId`
- `checkId`
- `questionId`
- `planId`
- 성취기준 코드
- 다음 점검일
- 데이터 버전

후속 도구 호출에 필요한 ID가 사용자 표시 본문에서도 누락되지 않아야 한다.

#### MCP 서버

- Streamable HTTP `/mcp`
- 상태 확인 `/health`
- Docker·Bun 기반 배포
- 교육과정 조회는 stateless
- 학부모 프로필·이력 기능은 인증된 사용자 범위에서 stateful
- 모든 도구에 PlayMCP annotation 적용

## 11. 경쟁 포지셔닝

### PlayMCP 내

직접 경쟁 MCP는 없다.

`학교생활 도우미`, `초·중·고 수업시간표`, `키즈허브`는 NEIS 기반 시간표·급식·학사일정·육아 정보를 제공하지만 교육과정 개념 그래프, 선수학습 판정, 자녀별 변화 추적은 제공하지 않는다.

### 핵심 차별점

- DECK이 보유한 초·중·고 교육과정 온톨로지
- 문제은행 가입 없이 학부모가 자연어로 사용
- 성취기준 코드와 학습 경로 근거 표시
- 점검 결과를 일회성 답변으로 끝내지 않고 자녀별로 누적
- 복습 계획·재점검·변화 리포트까지 하나의 흐름으로 제공
- AI School 벤치마크로 일반 AI의 교육과정 오류와 제품 필요성 설명

### 핵심 메시지

> AI가 공부를 가르치기 전에, 무엇을 어떤 순서로 배워야 하는지부터 정확히 알아야 합니다.
